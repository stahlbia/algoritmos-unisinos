PRIMEIRA VERSÃO DO PROJETO:

Faça um programa que armazene na mesma estrutura, seriados e filmes.
Os dados para armazenamento dos filmes e seriados são: código, nome do filme, ano, diretor, principais
atores, tipo (seriado ou filme).
Implemente as operações básicas da API de arrays (sim... nesta primeira versão, vamos usar arrays!)

SEGUNDA VERSÃO DO PROJETO:

Segue então mais alguns requisitos para vocês considerarem na implementação:
    • Filmes e seriados são produtos audiovisuais
    • Cada filme/seriado tem um único gênero
    • Exemplos de gêneros: ação, terror, ficção, comédia, entre outros.
    • Todo o filme/seriado pode fazer parte de mais de um streaming
    • Exemplos de streaming: Netflix, Amazon Prime, HBO Max, Star Plus, entre outros
    • Cada streaming possui categorias específicas como maratonar no fim de semana, top 10,
    exclusivo
    • Os filmes/seriados podem fazer parte de mais de uma categoria específica do streaming que
    estão associados
    • Além das informações anteriores (Hands on 2), os filmes possuem duração
    • Os seriados possuem temporadas e número de episódios por temporada
Exercício 1:
Implemente uma lista genérica para que possa armazenar qualquer estrutura necessária, com base nos
requisitos fornecidos
Exercício 2:
Crie uma estrutura de classes que considere os requisitos fornecidos acima, contemplando as regras
fornecidas.
Debata com o colega sobre práticas e melhores formas de fazer a manipulação.
Exercício 3:
Crie os testes de unidade necessários para as classes desenvolvidas
